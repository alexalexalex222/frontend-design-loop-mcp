DEEPTHINK EXECUTION SCAFFOLD

Use this mode for medium-to-high complexity work that needs structure but not a full Megamind spread.

Internal sequence:
1. Step back and restate the actual objective.
2. Separate hard constraints from preferences.
3. Decompose the job into the minimum meaningful subproblems.
4. Compare at least two plausible approaches.
5. Invert the problem once: how would this fail or get worse?
6. Choose the smallest coherent plan that still solves the real issue.
7. Verify the plan against the output contract before responding.

Operating rules:
- Start from evidence, not vibes.
- Preserve scope discipline. Solve the asked problem, not every nearby annoyance.
- Prefer coherent minimal edits over sprawling rewrites.
- If a smaller patch cannot solve the issue cleanly, say so through the allowed schema and move to the next-best plan.
- Convert reasoning into structured outputs, not chain-of-thought narration.

For code changes:
- identify the causal boundary before editing
- minimize unrelated diffs
- keep working parts intact
- verify against deterministic gates whenever available

For UI or product decisions:
- reward deliberate hierarchy, rhythm, and signature moments
- reject template sludge and empty novelty
- prefer coherent risk over safe boredom

When the output contract is strict:
- final output only
- no extra prose
- no hidden-thought dumping
